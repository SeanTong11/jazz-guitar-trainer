# 练耳工具需求

## 1. 背景

当前项目已经包含以下能力：

- 音名 / 级数练习
- 指板音阶可视化
- 和弦图表浏览
- 基于 `Tone` 的基础音频播放能力

结合 `纪要_现代爵士吉他指南_0328.md` 与 `现代爵士吉他指南_0328.srt` 中老师强调的训练重点，下一步适合在当前项目中新增一个通用型练耳模块，用于训练爵士语境下的音程、和弦、张力音与调式听觉。

该模块首版不做成某一首歌的专项工具，也不绑定 `So What`。`Dorian`、`So What`、`modal jazz` 相关内容应被视为练习场景或预设包，而不是产品本身的唯一目标。

长期目标是将该模块逐步演进为一个可替代 `My Ear Training` 的网页版练耳工具。

## 2. 产品定位

这是一个面向爵士学习者的通用型练耳工具，目标是帮助用户建立：

- 对基础音程和和弦色彩的听觉识别能力
- 对爵士和弦延伸音 `9 / 11 / 13` 的色彩感知
- 对常见调式与和弦类型之间关系的听觉理解
- 从“听到”过渡到“理解”和“弹奏”的练习闭环

首版重点仍然放在爵士和弦听辨，而不是覆盖所有练耳类别。

## 3. 产品目标

### 3.1 MVP 目标

首版应满足以下目标：

- 在当前项目中新增一个独立的练耳模块
- 用户可以直接在网页中开始一轮和弦听辨训练
- 练习过程包含播放、作答、反馈、统计四个完整环节
- 支持若干预设练习集，而不是要求用户自己从零配置
- 产品形态简单直接，能在桌面端和移动端正常使用

### 3.2 中期目标

- 扩展到音程、调式、旋律短句等练耳类型
- 支持自定义练习集与筛选规则
- 支持错题回练与阶段性统计
- 逐步靠近 `My Ear Training` 的覆盖范围

### 3.3 长期目标

- 形成一套完整的 Web 端练耳平台
- 支持练习模板保存、题库扩展、学习路径管理
- 成为当前项目中的核心训练模块之一

## 4. 用户与使用场景

目标用户：

- 爵士吉他学习者
- 需要训练和弦色彩识别的乐手
- 希望把调式、和弦、张力音听感和指板练习结合起来的用户

典型场景：

- 每日 10 到 20 分钟做和弦听辨
- 训练小七和弦及其延伸音色彩
- 针对某一类调式或和弦语境做集中训练
- 练耳后再回到指板模块和和弦图模块做联动练习

## 5. MVP 范围

### 5.1 首版练习类型

首版仅做 `Chord Ear Training`，不同时做全部练耳类别。

推荐首版支持以下练习集合：

- 基础三和弦：major / minor / diminished / augmented
- 七和弦：maj7 / 7 / m7 / m7b5 / dim7
- 爵士小和弦扩展：m7 / m9 / m11 / m6 或 m13

说明：

- `m6` 在爵士学习语境中可先作为 `minor 13` 的近似入口
- `Dorian` 相关的 `m7 / m9 / m11 / m13` 应作为重要预设集合，但不应成为唯一集合

### 5.2 首版交互流程

用户完成一轮题目的最小流程：

1. 选择练习集
2. 点击开始
3. 系统随机播放一个题目
4. 用户选择答案
5. 系统立即反馈对错与正确答案
6. 用户进入下一题或自动进入下一题
7. 页面持续显示正确数、错误数、正确率

### 5.3 首版题目形式

首版先采用选择题形式：

- 单题单答案
- 2 到 6 个选项
- 每题只有一个标准答案

首版暂不做：

- 输入式作答
- 键盘弹奏式作答
- 麦克风跟唱识别
- MIDI 输入判题

## 6. 功能需求

### 6.1 模块入口

需要在当前项目的主导航中新增一个模块，例如：

- `和弦练耳`
- 或 `Ear Training`

模块应与现有页面结构保持一致，作为独立 tab 存在。

### 6.2 练习配置

用户至少可以配置以下内容：

- 练习类别
- 题目集合
- 是否固定根音
- 是否随机根音
- 是否自动播放下一题

首版建议提供两层配置方式：

- 预设练习包
- 简单筛选项

不建议首版直接暴露过于复杂的配置界面。

### 6.3 预设练习包

首版建议内置以下预设：

- 基础三和弦辨识
- 基础七和弦辨识
- Minor Extensions
- Dorian Minor Extensions

后续可加入：

- Dominant Tensions
- Major 7 Tensions
- Modal Chords
- Rootless Voicings

### 6.4 音频播放

系统需要支持以下播放能力：

- 题目自动播放
- 用户手动重播
- 连续播放多题
- 统一的音量与包络设置

首版需要明确以下播放规则：

- 默认使用钢琴或合成器音色
- 每题先播放和弦整体
- 是否播放根音提示应可配置，或作为后续增强

后续可扩展的播放方式：

- 和弦整体播放
- 分解播放后再整体播放
- 根音先出再出和弦
- 上行 / 下行分解

建议作为下一阶段优先项：

- 增加“播放方式”配置，让用户可在整体播放与分解播放之间切换
- 在保持界面简单的前提下，把播放方式作为难度控制的一部分，而不是单独做成复杂编辑器

### 6.5 题目生成

题目生成逻辑至少需要支持：

- 从当前练习集合随机抽题
- 尽量避免连续重复同一题
- 在固定根音模式下只变化和弦类型
- 在随机根音模式下同时变化根音和和弦类型

后续可以加入：

- 按难度控制题目分布
- 根据用户错题动态提高重复概率
- 支持题库权重和自定义排除规则

### 6.6 作答与反馈

答题后需要立即给出反馈，至少包括：

- 是否正确
- 正确答案是什么
- 当前题目的构成说明

对于爵士和弦扩展类题目，建议显示：

- 和弦名称
- 组成音
- 相关 tension，如 `9 / 11 / 13`
- 对应常见语境，如 `Dorian` 可用等

### 6.7 统计与进度

首版应包含基础统计：

- 正确数
- 错误数
- 正确率
- 当前题号或当前轮次

后续可扩展：

- 按练习包统计
- 按和弦类型统计
- 错题本
- 历史训练记录
- 连续打卡

### 6.8 学习辅助信息

首版可以在答题反馈区或辅助信息区显示：

- 和弦构成音
- 与该和弦相关的常见乐理标签
- 适合继续去哪个模块复习

例如：

- 听完 `Dm11` 后，可引导用户去指板模块看 `D Dorian`
- 听完 `m11` 后，可引导用户去和弦图模块看 quartal 或 minor voicing

这部分不需要做得很重，但应保留联动空间。

建议作为后续增强：

- 在答题反馈中提供跳转入口，直接打开对应的指板或和弦图模块
- 优先做轻量联动，例如“去看指板”“去看和弦图”按钮，而不是一开始做复杂的跨模块状态同步

## 7. 非目标

以下内容不属于首版范围：

- 完整替代 `My Ear Training`
- 全量自定义和弦编辑器
- 用户账号系统
- 云端同步
- 麦克风识别
- MIDI 设备输入判题
- 旋律转录
- 节奏练耳
- 自动生成完整课程体系

这些能力可以进入后续版本规划，但不应影响首版上线。

## 8. 信息架构建议

建议新增一个独立模块：

- 现有模块
  - 音名 / 级数
  - 指板音阶
  - 和弦图表
- 新增模块
  - 和弦练耳

后续如果练耳能力继续扩展，可以演进为：

- 练耳首页
  - 和弦听辨
  - 音程听辨
  - 调式听辨
  - 旋律短句听辨

首版不需要过度设计这一层，但新模块命名与数据结构应为未来扩展留空间。

## 9. 结合当前项目的实现建议

基于当前仓库，首版可以复用以下能力：

- 现有模块导航结构
- 现有按钮、卡片、统计信息布局模式
- `Tone` 音频引擎
- 现有乐理常量与音阶数据

建议实现方式：

- 在 `src/modules/` 中新增一个练耳模块文件
- 在 `src/core/` 中补充和弦题库、和弦构成音与播放逻辑
- 在主导航中新增入口
- 复用当前项目的 UI 风格，不重新设计整套界面

如果后续真的要向 `My Ear Training` 替代品演进，应尽早把以下内容拆清楚：

- 练习配置数据结构
- 题目生成器
- 音频播放策略
- 统计存储结构
- 可扩展题库格式

## 10. 建议的版本路线

### v1

目标：

- 做出一个可用的和弦练耳模块

范围：

- 基础和弦听辨
- Minor Extensions 听辨
- 播放、作答、反馈、统计闭环

### v1.5

目标：

- 提升练习效率和可用性

范围：

- 预设练习包扩展
- 错题回练
- 难度切换
- 播放模式配置
- 整体播放 / 分解播放切换
- 反馈区到指板 / 和弦图模块的轻量联动

### v2

目标：

- 向 `My Ear Training` 替代品靠近

范围：

- 支持更多练耳类别
- 支持自定义练习集
- 支持本地保存训练记录
- 提供更完整的学习反馈

## 11. 待确认问题

在进入开发前，后续还需要继续确认以下问题：

- 首版是否只做和弦听辨，还是同时纳入音程听辨
- 首版默认是固定根音还是随机根音
- `m6` 是否在 UI 上直接标注为 `m6`，还是标注为 `m13`
- 首版是否需要显示构成音和 tension 提示
- 首版是否需要和现有指板 / 和弦图模块做显式联动入口

## 12. 结论

本次新增模块应被定义为：

一个面向爵士学习者的通用型 Web 练耳工具的第一步。

首版以和弦听辨为核心，重点覆盖爵士学习中最常见、最实用的一组训练任务，尤其是小七和弦及其延伸色彩。产品设计上不绑定单首曲目，不做专项工具，而是为后续逐步演进为 `My Ear Training` 替代品打基础。
